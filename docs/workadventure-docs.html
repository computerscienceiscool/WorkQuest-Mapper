Follow this quick-start guide to create your first WorkAdventure map in under 5 minutes.

üåê Deployment Guide

Option 1: GitHub Pages (Recommended for Beginners)

Create GitHub Repository

Sign up at GitHub and create a new public repository named "workadventure-map"

Upload Generated Files

Drag and drop your map.json, tileset.json, and map.js files into the repository

Enable GitHub Pages

Go to Settings ‚Üí Pages ‚Üí Source: "Deploy from a branch" ‚Üí Select "main" branch

Get Your Map URL

Your map will be available at: https://username.github.io/workadventure-map/map.json

Test the JSON File

When you visit the URL, you'll see raw JSON text (not a webpage) - this is correct! The JSON content should be properly formatted, not a jumbled string.

‚ö†Ô∏è Important: When you visit your map.json URL directly, you should see properly formatted JSON with line breaks and indentation, NOT a compressed single line of text. If you see jumbled text like "compressionlevel-1width40height30...", the JSON may be corrupted or improperly formatted.
üîç What You Should See: A properly formatted JSON file with clear structure, indentation, and readable property names. The raw JSON is what WorkAdventure needs - you're not meant to view it as a regular webpage.

üé® Custom Visual Assets

Update Tileset JSON

Modify tileset.json to reference your custom image file and define collision properties for each tile.

Ensure wall tiles have "collides: true" property to prevent players from walking through them.

üîß Troubleshooting

Common Issues & Solutions

‚ùå Map Won't Load

Symptoms: Blank screen or loading error
Solution: Check CORS headers, verify JSON syntax, ensure files are publicly accessible

üé• Jitsi Not Working

Symptoms: No video call when entering meeting rooms
Solution: Verify room names are unique, check browser permissions, test Jitsi independently

üñºÔ∏è Tiles Not Displaying

Symptoms: Missing graphics or broken visuals
Solution: Ensure tileset.json references correct image file, check image file permissions

üö´ Collision Issues

Symptoms: Players walk through walls
Solution: Verify "collides: true" property on wall tiles in tileset.json

Debug Checklist

Check Browser Console

Open F12 developer tools and look for JavaScript errors or network request failures

Validate JSON Syntax

Use JSONLint to verify your map.json and tileset.json files are properly formatted

Test File Accessibility

Directly access your map.json URL in a browser to ensure it loads without errors

Verify WorkAdventure Configuration

Ensure your WorkAdventure world settings point to the correct map URL

‚ö†Ô∏è Common Mistake: Mixing HTTP and HTTPS URLs. WorkAdventure requires HTTPS for security - ensure all your map files are served over HTTPS.
